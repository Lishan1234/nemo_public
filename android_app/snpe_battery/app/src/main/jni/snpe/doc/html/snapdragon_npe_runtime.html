Snapdragon NPE Runtime


Overview

This drawing describes some of the components of Snapdragon NPE Runtime Library which run on the device.
For details on using the library, see the tutorials and examples section : Tutorials Setup

At a high level, the library contains the following :

DL Container Loader : loads a DLC created by one of snpe-framework-to-dlc conversion tools.

Model Validation : validates that the loaded DLC is supported by the required runtime. See Supported Network Layers

Runtime Engine : Executes a loaded model on requested runtime/s, including gathering profiling info and supporting UDLs (see User-Defined Layers (UDL) Tutorial).

Partitioning Logic : Processes the model, including UDL and validation information, and creates partitioning of the model to sub-models if needed
For UDLs, breaks execution before and after the UDL to allow the UDL execution.
If CPU fallback is enabled, partitions the model between layers supported by the target runtime, and layers supported by the CPU runtime

CPU Runtime Runs the model on the CPU, support 32-bit floating point or 8-bit quantized execution.
GPU Runtime Runs the model on the GPU, supports a hybrid or full 16-bit floating point modes.
DSP Runtime Runs the model on Hexagon DSP using Q6 and Hexagon NN, executing on HVX.
AIP Runtime Runs the model on Hexagon DSP using Q6, Hexagon NN, and HTA
